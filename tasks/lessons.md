# 本地资产与知识库映射经验沉淀

## 目录重构策略
- **[触发动机]** 发现用户 `Y:\GGbommer\scripts` 目录下存在大量复杂的混合代码库（如既有 C++ 也有 Python 的 Maya 插件），按单纯后缀移动会破坏工程。
- **[根因分析]** 很多高价值工具（如 MetaHuman 工具链或 Maya C++ API 节点）是按“项目”进行组织的，而不是按“语言”。
- **[正确方案]** 放弃物理文件的转移，采用**知识库原地打标索引（Index-in-Place）**。让 Obsidian 作为只读仪表盘，通过 `file:///` 协议进行跳转，彻底解耦“物理存放”与“逻辑呈现”。
- **[避坑规则]** 绝对禁止使用机械的后缀名正则分类法处理影视 CGI/VFX 级复杂管线工具库；所有识别必须基于 `README`、依赖配置或多文件类型组合判断。
